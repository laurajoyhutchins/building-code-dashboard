# building-code-dashboard

Dashboard + ingestion pipeline for U.S. building code adoption data.

## Current Repository Structure

```text
building-code-dashboard/
|-- index.html
|-- building-code-dashboard.html
|-- README.md
|-- .claude/
|   `-- settings.local.json
`-- src/
    `-- py/
        |-- __init__.py
        |-- orchestrator.py
        |-- schema.py
        |-- icc_chart_parser.py
        |-- nec_scraper.py
        |-- iecc_scraper.py
        |-- municipal_scraper.py
        |-- test_suite.py
        |-- db/
        |   |-- __init__.py
        |   `-- schema.py
        |-- scrapers/
        |   |-- __init__.py
        |   |-- icc_chart_parser.py
        |   |-- nec_scraper.py
        |   |-- iecc_scraper.py
        |   `-- municipal_scraper.py
        |-- tests/
        |   |-- __init__.py
        |   `-- test_suite.py
        `-- __pycache__/ (generated)
```

## Notes

- `index.html` is the GitHub Pages entrypoint.
- `building-code-dashboard.html` is an additional HTML page copy/variant.
- Python scraper modules exist in both `src/py/` and `src/py/scrapers/`.
- `orchestrator.py` imports scrapers from `src/py/scrapers/`.
- Database schema files exist in both `src/py/schema.py` and `src/py/db/schema.py`.
- `__pycache__/` directories contain generated bytecode.

## Running From Repo Root

```bash
# Run pipeline
python src/py/orchestrator.py

# Optional flags
python src/py/orchestrator.py --include-municipal
python src/py/orchestrator.py --export-only

# Run tests
python src/py/test_suite.py
python -m src.py.tests.test_suite
```
